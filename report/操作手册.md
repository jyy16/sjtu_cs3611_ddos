# CS3611 Project 9 — 操作手册

---

## 1. 文档目的

本手册说明 **环境配置**、**依赖安装** 与 **脚本运行顺序**，供实验部署与演示复现使用。


---

## 2. 环境要求

| 项目   | 要求                      | 说明                    |
| ---- | ----------------------- | --------------------- |
| 操作系统 | Ubuntu 20.04+ 或其他 Linux | 需 `bash`、`sudo`、`tcpdump`、`iptables` |
| Python | 3.8+（推荐 3.10）           | 使用项目内 `.venv`       |
| 网络   | loopback / 私网           | 默认 `127.0.0.1`，禁止公网目标 |
| Redis  | 可选                      | 启用 `STORAGE_BACKEND=redis` 时需要 |

---

## 3. 环境配置步骤

### 步骤 1：获取项目代码

```bash
cd /path/to/sjtu_cs3611_ddos-auto_block
```

### 步骤 2：安装系统依赖

```bash
sudo apt update
sudo apt install -y python3 python3-pip python3-venv \
  tcpdump iptables curl bash git
```

| 包名                | 用途              |
| ----------------- | --------------- |
| `python3` / `pip` | 运行攻击、特征、模型、防御脚本 |
| `tcpdump`         | PCAP 抓包         |
| `iptables`        | 防火墙限速与黑名单       |
| `curl`            | 检查受害 HTTP 服务    |

### 步骤 3：安装 Python 依赖

推荐在 Linux/WSL 环境运行完整 demo。

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -r models/requirements.txt
```

在 `scripts/demo.env` 中确认：

```bash
PYTHON=.venv/bin/python
```

验证：

```bash
.venv/bin/python -c "import torch, pandas, sklearn; print('OK')"
```

### 步骤 4：演示配置

```bash
cp scripts/demo.env.example scripts/demo.env
```

确认 `TARGET_IP=127.0.0.1`（**切勿设为公网 IP**）。

### 步骤 5：Redis 存储（可选）

快速本地跑通可保持 `STORAGE_BACKEND=none`。

启用 Redis 时：

```bash
docker run --rm --name cs3611-redis -p 6379:6379 redis:7
redis-cli -u redis://127.0.0.1:6379/0 ping
```

修改 `scripts/demo.env`：

```bash
STORAGE_BACKEND=redis
REDIS_URL=redis://127.0.0.1:6379/0
STORAGE_KEY_PREFIX=cs3611:ddos
STORAGE_FAIL_OPEN=0
```

## 4. 脚本运行顺序

### 4.1 流程概览

```text
① 环境配置（§3，仅首次）
        ↓
② 三组合同自检（建议）
        ↓
③ 整合预检（--check-only）
        ↓
④ 普通演示 run_demo.sh
   和/或 实时演示 run_realtime_demo.sh（选做）
        ↓
⑤ 查看产出 / 生成可视化
        ↓
⑥ 清理防火墙规则
```

### 4.2 合同自检

```bash
bash scripts/check_group_contract.sh attack
bash scripts/check_group_contract.sh defense
bash scripts/check_group_contract.sh model
```

### 4.3 整合预检

```bash
bash scripts/run_demo.sh --check-only
```

### 4.4 普通演示

```bash
bash scripts/run_demo.sh --run-id report_demo_01
```

演示前重新训练模型（可选）：

```bash
bash scripts/run_demo.sh --train --run-id train_demo_01
```

`run_demo.sh` 内部顺序：

| 序号 | 阶段 | 说明 |
| --- | --- | --- |
| 1 | Preflight | 检查文件、命令、Redis（若启用） |
| 2 | 启动受害服务 | `http.server 8080`（`START_TARGET=1` 时自动启动） |
| 3 | 清空旧规则 | `unblock_all.sh` |
| 4 | 阶段 A | 正常流量抓包 → 特征 CSV |
| 5 | 阶段 B | 混合攻击抓包 → 特征 CSV → 推理 → 防御 |
| 6 | 阶段 C | 同参混合攻击抓包 → 特征 CSV |
| 7 | 收尾 | 输出 PCAP/CSV/日志路径 |

### 4.5 实时攻防演示

```bash
bash scripts/run_realtime_demo.sh --run-id realtime_demo_01
```

实时脚本在攻击进行时，后台按窗口执行「抓包 → 特征 → 推理 → 封禁」。

### 4.6 演示后清理

```bash
bash defense/unblock_all.sh --project-tag cs3611-ddos
```

停止 Docker Redis（若曾启动）：

```bash
docker stop cs3611-redis
```

## 5. 演示产出

材料默认在：

```text
data/pcap/<run_id>/
data/features/<run_id>/
data/logs/<run_id>/
```

本报告示例 Run ID：`report_demo_01`（普通）、`realtime_demo_01`（实时）。

```bash
ls -lh data/logs/report_demo_01/
cat data/logs/<run_id>/decision_<run_id>.json
```

生成可视化（可选）：

```bash
.venv/bin/python scripts/visualize_demo.py --run-id report_demo_01
.venv/bin/python scripts/visualize_realtime_demo.py --run-id realtime_demo_01
```

Redis 数据检查见 `docs/demo_commands.md` §6。

## 6. 常见问题

如果 `run_demo.sh --check-only` 报 Redis 依赖缺失：

```bash
.venv/bin/python -m pip install -r models/requirements.txt
```

如果 Redis 没启动：

```bash
redis-cli -u redis://127.0.0.1:6379/0 ping
```

如果本地抓不到 loopback 流量：

```bash
CAPTURE_IFACE=any bash scripts/run_demo.sh --run-id any_capture_test
```

如果 `tcpdump: permission denied`，在项目根目录使用 `sudo` 运行演示脚本。
